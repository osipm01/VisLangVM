# Документация системы управления памятью

## Обзор

Двухуровневая система управления памятью, реализующая арен-based аллокацию с поддержкой дефрагментации. Система обеспечивает типобезопасное управление памятью для JavaScript-значений с симуляцией автоматической сборки мусора через отслеживание свободных блоков.

## Архитектура

Система состоит из двух основных классов:
- **Arena**: Низкоуровневый пул памяти с управлением блоками
- **MemoryAllocator**: Высокоуровневый типобезопасный аллокатор, управляющий несколькими аренами

## Класс Arena

### Конструктор
```javascript
new Arena(capacity = 1024)
```
Создает новую арену с указанной вместимостью (в 32-битных единицах).

### Структура памяти

Каждый выделенный блок имеет следующую структуру:
```
Смещение+0: Тег типа (1 байт, закодированный как Int32)
Смещение+1: Метаданные/Длина
Смещение+2...: Данные
```

Кодировка типов:
- `1`: Строка (string)
- `2`: Число (number)
- `3`: Булево (boolean)
- `4`: Null
- `5`: Массив (array)

### Основные методы

#### Выделение памяти
```javascript
allocate(name, value)
```
Выделяет память для значения с указанным именем. Использует стратегию первого подходящего:
1. Ищет свободный блок подходящего размера
2. Если найден, использует его (разделяя, если блок больше)
3. Иначе добавляет в конец
4. Возвращает смещение или -1 при неудаче

#### Освобождение памяти
```javascript
free(name)
```
Освобождает память, занятую переменной с указанным именем:
- Очищает область памяти
- Добавляет блок в список свободных
- Объединяет соседние свободные блоки
- Обновляет lastIndex при освобождении в конце

#### Обновление
```javascript
set(name, value)
```
Обновляет существующую переменную с автоматическим перемещением при увеличении размера:
- Если новый размер ≤ старого: обновляет на месте, добавляет остаток как свободный блок
- Если новый размер > старого: освобождает старую позицию, выделяет новую
- Откатывает изменения при неудаче выделения

#### Чтение
```javascript
get(name)
```
Получает значение по имени, автоматически декодируя тип и восстанавливая исходное значение.

### Управление памятью

**Отслеживание свободных блоков**
- Поддерживает отсортированный список свободных блоков
- Реализует объединение блоков при освобождении
- Стратегия выделения "первый подходящий"

**Дефрагментация**
- Автоматическое разделение блоков при выделении
- Объединение смежных свободных блоков
- Обрезка при освобождении в конце

## Класс MemoryAllocator

### Конструктор
```javascript
new MemoryAllocator()
```
Создает тип-специфичные арены с размерами по умолчанию:
- Арена строк: 2048 единиц
- Арена чисел: 1024 единицы
- Арена булевых: 512 единиц
- Арена null: 128 единиц
- Арена массивов: 4096 единиц

### Публичные методы

#### Выделение
```javascript
allocate(name, value)
```
Автоматически определяет тип значения и выделяет память в соответствующей арене.

#### Доступ
```javascript
get(name)
```
Получает значение из правильной арены.

#### Обновление
```javascript
set(name, value)
```
Обновляет значение, обрабатывая изменение типа миграцией между аренами при необходимости.

#### Освобождение
```javascript
free(name)
```
Удаляет переменную из реестра и арены.

#### Управление
```javascript
listVariables()
```
Возвращает массив {name, type, value} для всех выделенных переменных.

```javascript
getStatistics()
```
Возвращает статистику использования по типам арен:
```javascript
Map {
  'string' => { used: 42, total: 2048, percentage: 2 },
  'number' => { used: 128, total: 1024, percentage: 12.5 },
  ...
}
```

## Примеры использования

### Базовые операции
```javascript
const allocator = new MemoryAllocator();

// Выделение разных типов
allocator.allocate("username", "john_doe");
allocator.allocate("score", 100);
allocator.allocate("active", true);
allocator.allocate("data", new Int32Array([1, 2, 3, 4]));

// Чтение значений
console.log(allocator.get("username")); // "john_doe"

// Обновление значений
allocator.set("score", 200); // Тот же размер, обновление на месте
allocator.set("username", "very_long_username"); // Перемещение при необходимости

// Освобождение памяти
allocator.free("temp_data");
```

### Миграция типов
```javascript
// Автоматическое перемещение между аренами при изменении типа
allocator.allocate("counter", 42); // Хранится в арене чисел
allocator.set("counter", "forty-two"); // Мигрирует в арену строк
```

### Мониторинг
```javascript
// Получение статистики
const stats = allocator.getStatistics();
for (const [type, {used, total, percentage}] of stats) {
  console.log(`${type}: ${used}/${total} (${percentage}%)`);
}

// Список всех переменных
console.log(allocator.listVariables());
```

## Эффективность использования памяти

### Накладные расходы на блок
- Минимальный размер блока: 2 единицы (тег типа + метаданные)
- Строки: 2 + ceil(длина/4) единиц
- Массивы: 2 + длина единиц

### Контроль фрагментации
- Стратегия выделения "первый подходящий"
- Автоматическое объединение блоков при освобождении
- Обновление на месте когда возможно
- Перемещение только при необходимости

## Обработка ошибок

Система обеспечивает корректную обработку ошибок:
- Возврат -1 из allocate() при неудаче
- Возврат false из операций при неудаче
- Логирование информативных сообщений об ошибках
- Возможность отката в операции set()

## Ограничения

- Фиксированная вместимость арен (настраивается)
- Нет автоматической дефрагментации между аренами
- Только 32-битные целочисленные массивы (нет чисел с плавающей точкой)
- Кодировка строк ограничена диапазоном Latin-1 (0-255)
- Нет сборки мусора - требуется явный вызов free()

## Производительность

- Выделение: O(n), где n - количество свободных блоков
- Освобождение: O(n log n) из-за сортировки
- Получение: O(1)
- Обновление: O(n) в худшем случае из-за перемещения
- Дефрагментация: автоматическая при освобождении
